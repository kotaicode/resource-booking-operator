Resource Booking Operator

Cloud resource scheduling using Kubernetes custom resources

Easily start cloud resources such as instances and databases when you need them, and stop them automatically to reduce cloud cost.

You don't need your databases at night? Create a schedule to stop them over night, and start them in the morning.

You need your BI instance only when the big reporting is due? Only start it on demand, make sure it is off when not needed.

Easily accessible directly from your kubernetes controlplane, leveraging the extensibility of the Kubernetes API

memory

Automatically start and stop cloud resources by using the custom Booking type in your Kubernetes cluster.

savings

Reduce cloud operating costs by using on-demand scheduling of resources. Stops resources when they are no longer required.

extension
